### Question 1 — Master Theorem application to a divide-and-conquer recurrence (asymptotic Theta bound).

**Question.** Which of the answers below holds for the following recurrence (rekursionsligning)? $T(n) = 5 \cdot T(n/2) + n^2$. [2.5 points]

**Options.**
0. $T(n)=\Theta(n^p)$ with $p=\log_2(5)$. ✓ **$T(n)=\Theta(n^p)$ with $p=\log_2(5)$.**
1. $T(n)=\Theta(n^p)$ with $p=\log_5(2)$.
2. $T(n)=\Theta(n^p \log n)$ with $p=\log_2(5)$.
3. $T(n)=\Theta(n^p \log n)$ with $p=\log_5(2)$.
4. $T(n)=\Theta(n^2)$.
5. The recurrence cannot be solved with the Master Theorem.

**Answer.** alternative 0: $T(n)=\Theta(n^p)$ with $p=\log_2(5)$.

**Method (reusable).** 1. Identify a, b, f(n) in T(n)=a·T(n/b)+f(n). 2. Compute the critical exponent p = log_b(a). 3. Compare f(n) with n^p. 4. Case 1: if f(n)=O(n^{p-eps}) then T(n)=Theta(n^p). Case 2: if f(n)=Theta(n^p) then T(n)=Theta(n^p log n). Case 3: if f(n)=Omega(n^{p+eps}) and regularity holds, then T(n)=Theta(f(n)). 5. Pick the matching alternative.

**Worked solution.** a=5, b=2, f(n)=n^2. Critical exponent p=log_2(5)≈2.3219. Compare f(n)=n^2 with n^{2.3219}: since 2 < log_2(5), f(n)=O(n^{p-eps}) for some eps>0. This is Master Theorem Case 1, so T(n)=Theta(n^{log_2 5}). Matches alternative 0.

### Question 2 — Asymptotic analysis of a divide-and-conquer recurrence via the Master Theorem.

**Question.** 2.5 points. Which of the answers below holds for the following recurrence (rekursionsligning)? $T(n) = 5\cdot T(n/5) + n\log n$.

**Options.**
0. $T(n)=\Theta(n^p)$ with $p=\log_5(5)$.
1. $T(n)=\Theta(n^p\log n)$ with $p=\log_5(5)$.
2. The recurrence cannot be solved with the Master Theorem (Rekursionsligningen kan ikke loses med Master Theorem). ✓ **The recurrence cannot be solved with the Master Theorem.**

**Answer.** alternative 2: The recurrence cannot be solved with the Master Theorem.

**Method (reusable).** 1. Identify a, b, f(n). 2. Compute p=log_b(a), form n^p. 3. Compare f(n) with n^p across the three Master Theorem cases. 4. If f and n^p differ only by a polylog factor, the standard theorem does not apply. 5. Match against options; choose 'cannot be solved' when no closed form fits and a gap case holds.

**Worked solution.** a=5, b=5, f(n)=n log n. p=log_5(5)=1, so n^p=n. Compare f(n)=n log n to n: f exceeds n^p only by a factor log n, which is not polynomial. Case 1 fails (f is not O(n^{1-eps})). Case 2 fails (f is not Theta(n), it has an extra log). Case 3 fails (no eps>0 with f=Omega(n^{1+eps})). The recurrence sits in the gap between case 2 and case 3, so the standard Master Theorem gives no answer. (The extended version yields Theta(n log^2 n), which is not among the options.) Alternative 0 = Theta(n) and alternative 1 = Theta(n log n) are both wrong. Correct: alternative 2.

### Question 3 — Master Theorem applied to a divide-and-conquer recurrence; determine the asymptotic Θ-bound.

**Question.** Which of the answers below holds for the following recurrence (rekursionsligning)? T(n) = 2·T(n/5) + n^{1/2}

**Options.**
0. T(n) = Θ(n^p) with p = log_2(5).
1. T(n) = Θ(n^p) with p = log_5(2).
2. T(n) = Θ(n^p log n) with p = log_2(5).
3. T(n) = Θ(n^p log n) with p = log_5(2).
4. T(n) = Θ(n^{1/2}). ✓ **T(n) = Θ(n^{1/2}).**
5. The recurrence cannot be solved with the Master Theorem.

**Answer.** alternative 4: T(n) = Θ(n^{1/2}).

**Method (reusable).** 1. Read off a, b, f(n) from T(n)=a·T(n/b)+f(n). 2. Compute p = log_b(a). 3. Compare f(n)=n^c against n^p. 4. c<p → case 1 Θ(n^p); c=p → case 2 Θ(n^p log n); c>p with regularity → case 3 Θ(f(n)). 5. Match to options.

**Worked solution.** a=2, b=5, f(n)=n^{1/2} so c=0.5. p = log_5(2) ≈ 0.4307. Since c=0.5 > p≈0.43, this is Master Theorem case 3. Regularity check: a·f(n/b)=2·(n/5)^{1/2}=(2/√5)·n^{1/2}≈0.894·n^{1/2} ≤ c·f(n) for c<1, holds. Therefore T(n)=Θ(f(n))=Θ(n^{1/2}). This is alternative 4. (Note p=log_5(2), not log_2(5), ruling out 0 and 2; case 3 gives no log factor, ruling out 1 and 3; Master Theorem does apply, ruling out 5.)

### Question 4 — Master Theorem applicability / recurrence-solving (asymptotics)

**Question.** Which of the following answers holds for the following recurrence equation (rekursionsligning)? T(n) = 2 \cdot T(n-2) + n. [2.5 points]

**Options.**
0. T(n) = \Theta(n^p) with p = \log_2(2).
1. T(n) = \Theta(n^p \log n) with p = \log_2(2).
2. The recurrence equation cannot be solved with the Master Theorem (kan ikke løses med Master Theorem). ✓ **The recurrence equation cannot be solved with the Master Theorem.**

**Answer.** alternative 2: The recurrence equation cannot be solved with the Master Theorem.

**Method (reusable).** 1. Check the recurrence form. Master Theorem applies only to divide-and-conquer recurrences T(n)=a·T(n/b)+f(n), where the subproblem size is the input DIVIDED by b>1. 2. If the recurrence subtracts a constant (T(n-c)) instead of dividing, it is a subtract-and-conquer recurrence; the Master Theorem does not apply. 3. Such subtraction recurrences typically grow exponentially (Θ(a^(n/c))), not polynomially, so any Θ(n^p) answer is also wrong. 4. Conclude Master Theorem cannot be used.

**Worked solution.** The recurrence is T(n)=2·T(n-2)+n. The subproblem is n-2 (input minus a constant), not n/b. Master Theorem requires the divide form T(n)=a·T(n/b)+f(n) with b>1, so it does not apply here. Numerically the values 4,12,30,68,146,...,5242836 for n=2..40 grow exponentially, confirming T(n)=Θ(2^(n/2)) rather than polynomial, ruling out alternatives 0 (Θ(n^1)) and 1 (Θ(n log n)). Correct alternative is 2.

### Question 5 — Asymptotic notation / big-O comparison (multiple correct answers)

**Question.** Which of the following statements are true? [One or more answers.] (Hvilke af nedenstående udsagn er sande? [Et eller flere svar.]) (10 points)

**Options.**
0. $2^n$ is $O(n^3)$
1. $n^2$ is $O(3^n)$ ✓ **$n^2$ is $O(3^n)$**
2. $n(\log n)^2$ is $O(n^3 \log n)$ ✓ **$n(\log n)^2$ is $O(n^3 \log n)$**
3. $n^2 \log n$ is $O(n(\log n)^3)$
4. $n^3$ is $O(n^2)$
5. $3^n$ is $O(2^n)$
6. $n^{1/3}$ is $O(n^{1/2})$ ✓ **$n^{1/3}$ is $O(n^{1/2})$**

**Answer.** alternative 1: True statements: (1) $n^2$ is $O(3^n)$; (2) $n(\log n)^2$ is $O(n^3\log n)$; (6) $n^{1/3}$ is $O(n^{1/2})$. (Multiple-answer question; correctN reports the first true alternative, 1.)

**Method (reusable).** 1. For each claim f = O(g), test whether f/g stays bounded as n grows. 2. To avoid overflow, compute log(f_n/g_n) = log f_n - log g_n at several increasing n. 3. If the log-ratio increases without bound, f grows faster than g, so f is NOT O(g). If it stays flat or decreases, f IS O(g). 4. Use the ordering: constants < log powers < polynomial powers (compare exponents) < exponentials (compare bases). 5. Collect every alternative that is true.

**Worked solution.** Ordering rules and log-ratio checks: (0) 2^n vs n^3: exponential beats polynomial, log-ratio rises -> FALSE. (1) n^2 vs 3^n: polynomial < exponential -> TRUE. (2) n(log n)^2 vs n^3 log n: ratio = (log n)/n^2 -> 0 -> TRUE. (3) n^2 log n vs n(log n)^3: ratio = n/(log n)^2 -> infinity, log-ratio rises -> FALSE. (4) n^3 vs n^2: higher exponent -> FALSE. (5) 3^n vs 2^n: larger base -> FALSE. (6) n^(1/3) vs n^(1/2): 1/3 < 1/2 -> TRUE. True: 1, 2, 6.

### Question 6 — Heap operations trace (CLRS HEAP-INCREASE-KEY then HEAP-EXTRACT-MAX on a max-heap array)

**Question.** First perform HEAP-INCREASE-KEY(A,9,15) and then HEAP-EXTRACT-MAX(A) on the max-heap A below. A (positions 1..9) = [18, 9, 16, 4, 8, 12, 13, 1, 2]. Which of the answers below shows the heap after these two operations?

**Options.**
0. A = [16, 15, 13, 9, 8, 4, 12, 1]
1. A = [16, 15, 13, 9, 8, 12, 4, 1] ✓ **A = [16, 15, 13, 9, 8, 12, 4, 1]**
2. A = [16, 15, 13, 4, 8, 12, 2, 1]
3. A = [16, 15, 13, 4, 8, 12, 1, 2]
8. 3 er O(2)  [3 is O(2)]

**Answer.** alternative 1: A = [16, 15, 13, 9, 8, 12, 4, 1]

**Method (reusable).** 1. Use 1-indexed array; parent(i)=floor(i/2), left=2i, right=2i+1. 2. HEAP-INCREASE-KEY(A,i,key): set A[i]=key (only if larger), then bubble up while A[parent]<A[i], swapping. 3. HEAP-EXTRACT-MAX: save A[1] as max, move last element to A[1], decrement heap size, then MAX-HEAPIFY(A,1): repeatedly swap with the larger of the two children while a child exceeds it. 4. Match the resulting array against the alternatives.

**Worked solution.** Start: [18,9,16,4,8,12,13,1,2]. Increase-Key pos9 to 15: A[9]=15; parent pos4=4<15 swap; then parent pos2=9<15 swap; parent pos1=18>15 stop. Result [18,15,16,9,8,12,13,1,4]. Extract-Max: max=18, move last (4) to root, heapsize=8, MAX-HEAPIFY from root: 4 vs children 15,16 -> swap with 16; 4 vs children 12,13 -> swap with 13; 4 vs child 1 -> stop. Final: [16,15,13,9,8,12,4,1]. Matches alternative 1.

### Question 8 — Minimum spanning tree / Prim's algorithm trace (find last EXTRACT-MIN vertex)

All in O(log n).

**Method (reusable).** 1. Identify the tree: a balanced BST over the points keyed on one coordinate (here x), augmented with subtree xmin/xmax/ymin/ymax at every node. 2. A point on a vertical side has an extreme x (right = max x, left = min x); a point on a horizontal side has an extreme y (top = max y, bottom = min y). 3. The BST's own key is x, so Tree-Maximum/Tree-Minimum directly return the extreme-x nodes in O(log n) -> the vertical sides. 4. The non-key coordinate (y) is NOT sorted by the BST, so use the augmented ymax/ymin: a recursive search from the root that at each step descends toward the child whose subtree ymax (resp. ymin) equals the current node's ymax (resp. ymin), reaching the global extreme-y node in O(log n) -> the horizontal sides. 5. Reject any option using Tree-Max/Min for the non-key (y) extremes, and reject inorder traversal (O(n), not O(log n)).

**Worked solution.** Tree is x-ordered, augmented with xmin/xmax/ymin/ymax. Vertical sides need extreme x: since x is the BST key, Tree-Maximum gives the rightmost point and Tree-Minimum the leftmost, both O(log n). Horizontal sides need extreme y, which the x-ordering does not arrange; instead descend from the root following the child whose subtree ymax matches the node's ymax to reach the top point (and ymin for the bottom), each O(log n) thanks to the augmentation. Alt 1 states exactly this and is correct. Alt 0 wrongly uses Tree-Max/Min for the y (horizontal) extremes, which an x-keyed tree does not support. Alt 2 wrongly uses Tree-Max/Min for all four. Alt 4 uses inorder traversal = O(n). Alt 3 is also algorithmically valid (xmax/xmin-guided search reproduces Tree-Maximum/Minimum on the x key, and ymax/ymin-guided search gives the y extremes), but Alt 1 is the textbook-standard intended answer using Tree-Maximum/Minimum for the key dimension.

### Question 29 — Data structures (dynamic point set: bounding rectangle via min/max x and y, with O(1) area and O(log n) updates) — pick the structure that works

**Question.** We now want to achieve the above results — namely to find the area of the spanned (bounding) rectangle in $O(1)$ time, and to find four points, one on each side of the rectangle, in $O(\log n)$ time — in another way. Insertions and deletions must still be possible in $O(\log n)$ time. Assume no $x$-coordinate appears in two different points, and no $y$-coordinate appears in two different points. It is claimed this can be achieved using two trees with no extra information in any node. Which of the following data structures allow this?

**Options.**
0. One tree is a max-heap whose keys are the points' x-coordinates. The other tree is a max-heap whose keys are the points' y-coordinates.
1. One tree is a max-heap whose keys are the points' x-coordinates. The other tree is a red-black tree whose keys are the points' y-coordinates, storing the largest and smallest y-coordinate of the tree in two global variables.
2. One tree is a red-black tree whose keys are the points' x-coordinates, storing the largest and smallest x-coordinate in two global variables. The other tree is a red-black tree whose keys are the points' y-coordinates, storing the largest and smallest y-coordinate in two global variables. ✓ **One tree is a red-black tree keyed on x with min/max x cached in two global variables; the other is a red-black tree keyed on y with min/max y cached in two globals.**
3. One tree is a Huffman tree whose keys are the points' x-coordinates as frequencies. The other tree is a Huffman tree whose keys are the points' y-coordinates as frequencies.

**Answer.** alternative 2: One tree is a red-black tree keyed on x with min/max x cached in two global variables; the other is a red-black tree keyed on y with min/max y cached in two globals.

**Method (reusable).** 1. Identify the required operations: bounding-rectangle area in O(1), the four extreme points (min/max x, min/max y) in O(log n), insert/delete in O(log n). 2. Note min AND max are needed on both coordinates. 3. A heap gives only max (or only min) in O(1) and finding the other extreme or deleting an arbitrary point is not O(log n) — eliminate heap-based options. 4. A balanced BST (red-black tree) supports insert/delete and min/max (leftmost/rightmost) in O(log n). 5. To get area in O(1) you must cache the current min/max; maintaining global min/max variables on each red-black tree does this. 6. You need one tree keyed on x and one keyed on y to cover both coordinate extremes. 7. Huffman trees are static frequency-construction tools, irrelevant — eliminate. 8. Pick the option with two red-black trees (one per coordinate) plus cached min/max globals.

**Worked solution.** The rectangle's four sides are determined by min x, max x, min y, max y; the four boundary points are the points achieving these extremes. Area = (maxX - minX)*(maxY - minY), needed in O(1). Option 0: two max-heaps give only maxima; you cannot get the minima, and deleting an arbitrary point on update is not O(log n). Fails. Option 1: heap on x cannot deliver both min and max x in O(log n); only the y-tree is fixed. Fails. Option 3: Huffman trees encode frequencies and are built statically; they do not support coordinate range/extreme queries or dynamic insert/delete here. Fails. Option 2: a red-black tree keyed on x yields leftmost/rightmost (min/max x) in O(log n) and supports insert/delete in O(log n); same for y. Storing current min/max in global variables makes area O(1), and these globals are not per-node extra information. This satisfies every requirement, so the answer is alternative 2.
