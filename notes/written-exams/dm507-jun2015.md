## Opgave 1 (10%) — Master theorem

**Problem.** For each recurrence say (a) whether the textbook Master theorem applies, and (b) if so give T(n).
- i) T(n) = 5·T(n/2) + n²
- ii) T(n) = 5·T(n/5) + n log n
- iii) T(n) = 2·T(n/5) + n^(1/2)
- iv) T(n) = 2·T(n−2) + n

**Type.** Solve divide-and-conquer recurrences with the Master theorem.

**Method (plug in your own a, b, f(n)).**
1. The Master theorem only fits the form `T(n) = a·T(n/b) + f(n)` with constants a ≥ 1, b > 1. If the recurrence is not that shape (e.g. subtractive `T(n−c)`, or variable a/b), the theorem does not apply.
2. Compute the watershed exponent `c* = log_b(a)` and compare f(n) to `n^c*`:
   - Case 1: f(n) = O(n^(c*−ε)) for some ε>0 → T(n) = Θ(n^c*).
   - Case 2: f(n) = Θ(n^c*) (optionally × log) → T(n) = Θ(n^c* · log n).
   - Case 3: f(n) = Ω(n^(c*+ε)) for some ε>0 AND regularity (a·f(n/b) ≤ k·f(n), k<1) → T(n) = Θ(f(n)).
3. If f(n) sits between two cases by only a logarithmic factor (no polynomial gap), the theorem has a gap and does not apply.

**Worked solution.**
- i) a=5, b=2, c* = log₂5 ≈ 2.3219. f(n)=n², and 2 < c*, so f = O(n^(c*−ε)). **Case 1 applies.** T(n) = **Θ(n^(log₂5))**.
- ii) a=5, b=5, c* = log₅5 = 1, so n^c* = n. f(n)= n log n. f/n = log n, bigger than n by only a log factor, not polynomially — the Case-2/Case-3 gap. **Master theorem does NOT apply.**
- iii) a=2, b=5, c* = log₅2 ≈ 0.4307. f(n)=n^(1/2)=n^0.5, and 0.5 > c* by a polynomial margin; regularity holds (2·(n/5)^0.5 ≈ 0.894·n^0.5 < n^0.5). **Case 3 applies.** T(n) = **Θ(√n)**.
- iv) Subtractive form `2·T(n−2)+n`, not `a·T(n/b)`. **Master theorem does NOT apply.**

**Answer (a):** applies for i and iii; not for ii and iv.
**Answer (b):** i) Θ(n^(log₂5)); iii) Θ(√n). (checked: Python — log₂5 ≈ 2.3219 > 2 so case 1 for i; log₅2 ≈ 0.4307 < 0.5 with regularity const 2/√5 ≈ 0.894 < 1 for iii.)

## Opgave 2 (10%) — True/false O-claims

**Problem.** State true/false:
i) 2n = O(n³); ii) n² = O(3ⁿ); iii) n(log n)² = O(n³ log n); iv) n² log n = O(n(log n)³); v) n³ = O(n²); vi) 3ⁿ = O(2ⁿ); vii) n^(1/3) = O(n^(1/2)); viii) (1/3)ⁿ = O((1/2)ⁿ); ix) 3 = O(2).

**Type.** Decide asymptotic upper-bound claims.

**Method.** f = O(g) iff limsup f(n)/g(n) < ∞. Rank by growth class: constant ≪ logs ≪ polynomials (by exponent) ≪ exponentials (by base). For two exponentials aⁿ, bⁿ: aⁿ = O(bⁿ) iff a ≤ b. Constants are O of any positive constant.

**Worked solution.**
- i) **True.** Linear ≤ cubic.
- ii) **True.** Polynomial ≤ any exponential with base > 1.
- iii) **True.** ratio = (log n)/n² → 0.
- iv) **False.** ratio n² log n / (n(log n)³) = n/(log n)² → ∞ (at n=10⁶ ≈ 2517).
- v) **False.** n³ ≫ n².
- vi) **False.** 3ⁿ/2ⁿ = (3/2)ⁿ → ∞.
- vii) **True.** Exponent 1/3 ≤ 1/2.
- viii) **True.** (1/3)ⁿ/(1/2)ⁿ = (2/3)ⁿ → 0.
- ix) **True.** Both constant.

**Answer:** i T, ii T, iii T, iv F, v F, vi F, vii T, viii T, ix T. (checked: Python — iv ratio n/(log n)² at n=10⁶ ≈ 5239 and growing, so false.)

## Opgave 3 (7%) — Heap operations

**Problem.** Max-heap A (array, 1-indexed): `A = [18, 9, 16, 4, 8, 12, 13, 1, 2]`. First Heap-Increase-Key(A, 9, 15), then Heap-Extract-Max(A). Give the array after each.

**Type.** Trace binary-heap Increase-Key and Extract-Max.

**Method.**
1. Increase-Key(A, i, key): set A[i]=key, then sift up — while i>1 and A[parent(i)] < A[i], swap with parent, move up. parent(i)=⌊i/2⌋.
2. Extract-Max: save A[1]; move last element to root; shrink size by 1; Max-Heapify(A, 1) — repeatedly swap root with its larger child until the heap property holds.

**Worked solution.**
- Increase-Key(A, 9, 15). Position 9 holds 2; set it to 15. Sift up: parent(9)=4 holds 4, 15>4 → swap. At pos 4; parent(4)=2 holds 9, 15>9 → swap. At pos 2; parent(2)=1 holds 18, 15<18 → stop.
  Result: **[18, 15, 16, 9, 8, 12, 13, 1, 4]**.
- Extract-Max returns **18**. Move last (4) to root → [4,15,16,9,8,12,13,1], size 8. Max-Heapify(1): children 15,16 → largest 16 → swap → [16,15,4,9,8,12,13,1]. At pos3: children 12,13 → largest 13 → swap → [16,15,13,9,8,12,4,1]. Done.
  Result: **[16, 15, 13, 9, 8, 12, 4, 1]**. (checked: Python)

## Opgave 4 (7%) — Quadratic probing

**Problem.** Table H of size 11, quadratic probing, h′(x) = (3x+5) mod 11, c₁=3, c₂=1, so h(x,i) = (h′(x) + 3i + i²) mod 11. Initial: index0=13, 1=39, 3=36, 8=23, 9=5 (rest empty). Insert 22, 16, 17 in that order; show table after each (empty = x).

**Type.** Trace open-addressing insert with quadratic probing.

**Method.** For each key x and probe i=0,1,2,…: slot = (h′(x) + c₁·i + c₂·i²) mod m. First empty slot wins.

**Worked solution.**
- Insert 22: h′(22)=71 mod 11=5. i=0 → slot 5 empty → place.
  `[13, 39, x, 36, x, 22, x, x, 23, 5, x]`
- Insert 16: h′(16)=53 mod 11=9. i=0 → 9 occupied. i=1 → (9+3+1) mod 11=2 empty → place.
  `[13, 39, 16, 36, x, 22, x, x, 23, 5, x]`
- Insert 17: h′(17)=56 mod 11=1. i=0 → 1 occ. i=1 → 5 occ. i=2 → (1+6+4)=11 mod 11=0 occ. i=3 → (1+9+9)=19 mod 11=8 occ. i=4 → (1+12+16)=29 mod 11=7 empty → place.
  `[13, 39, 16, 36, x, 22, x, 17, 23, 5, x]` (checked: Python)

## Opgave 5 (7%) — Prim's algorithm

**Problem.** Undirected weighted graph, start at a. Edges: a–b=17, a–e=11, a–d=8, b–e=5, e–d=18, b–c=14, e–c=3, c–d=25. Give the Extract-Min order and the MST edges.

**Type.** Trace Prim, report extraction order and MST.

**Method.** key[a]=0, rest ∞, π=NIL. Repeatedly Extract-Min the lowest-key vertex u, output it, add edge (u, π[u]) (if π[u]≠NIL). For each neighbor v still in queue with w(u,v) < key[v], set key[v]=w(u,v), π[v]=u.

**Worked solution.**
| step | extracted (key) | updates after |
|------|------|------|
| 1 | a (0) | b←17, e←11, d←8 |
| 2 | d (8) | c←25 (e:18 not < 11) |
| 3 | e (11) | b←5, c←3 |
| 4 | c (3) | (b:14 not < 5) |
| 5 | b (5) | — |

**Extract-Min order: a, d, e, c, b.**
**MST edges: (a,d)=8, (a,e)=11, (e,c)=3, (e,b)=5.** Total weight 27. (checked: Python)

## Opgave 6 (10%) — DFS times and edge types

**Problem.** "Wheel" graph: center i, outer ring a–b–c–d–e–f–g–h–a (clockwise from top), every ring vertex also joined to i. Edges undirected. DFS starts at i, adjacency lists in alphabetical order.
- a) Give v.d (discovery) and v.f (finish) for all vertices.
- b) Classify every edge as tree / back / forward / cross (endpoints alphabetical).

**Type.** Trace DFS, assign timestamps, classify edges.

**Method.**
1. Clock from 0. Entering v: time++, v.d=time, grey. Recurse into white neighbors in adjacency order (alphabetical), making tree edges. Finishing v: black, time++, v.f=time.
2. Edge (u,v) from u: v white → tree; v grey → back; v black → forward if u.d<v.d else cross. For undirected graphs only tree and back edges occur.

**Worked solution (a).** From i (d=1), neighbors a,b,c,d,e,f,g,h; first white = a (tree i–a). From a, neighbors b,h,i; first white b → ring walks a→b→c→d→e→f→g→h, each a tree edge; h finishes first.

| v | d | f |
|---|---|---|
| i | 1 | 18 |
| a | 2 | 17 |
| b | 3 | 16 |
| c | 4 | 15 |
| d | 5 | 14 |
| e | 6 | 13 |
| f | 7 | 12 |
| g | 8 | 11 |
| h | 9 | 10 |

**(b) Edge types** (undirected → only tree and back).
- **Tree:** (a,i), (a,b), (b,c), (c,d), (d,e), (e,f), (f,g), (g,h).
- **Back:** (a,h), (b,i), (c,i), (d,i), (e,i), (f,i), (g,i), (h,i). (Ring-closing a–h plus every spoke to i except the tree spoke a–i.)
- **Forward:** none.
- **Cross:** none.

## Opgave 7 (10%) — Which shortest-path algorithm, and runtimes

**Problem (a).** Three graphs share a topology (source-like c, sink-like d): edges b→a, a→e, a→d, b→e, c→b, c→e, e→d, plus one bottom-right edge that differs. Mark with × which of Dijkstra (D), Bellman-Ford (BF), DAG-Shortest-Paths (DAG), BFS, DFS correctly compute shortest paths.
- G1: all weights = 1.
- G2: b→a=1, a→e=1, a→d=−1, b→e=−1, c→b=1, c→e=−1, e→d=1, **c→d=1**. DAG with negative edges, no negative cycle.
- G3: same as G2 except the bottom-right edge is **d→c=1** (instead of c→d). This creates the directed cycle a→d→c→b→a (weight +2). Negative edges present, but every cycle is positive (no negative cycle).

**Type.** Match algorithm preconditions to graph properties.

**Method.** Check: (1) all weights ≥ 0? (2) acyclic (DAG)? (3) all weights equal (so BFS=shortest path)?
- D: needs nonnegative weights.
- BF: works with negatives if no negative-weight cycle.
- DAG: needs acyclic.
- BFS: correct only when all edge weights are equal.
- DFS: never computes shortest paths — never marked.

**Worked solution.**

|    | D | BF | DAG | BFS | DFS |
|----|---|----|-----|-----|-----|
| G1 | × | ×  | ×   | ×   |     |
| G2 |   | ×  | ×   |     |     |
| G3 |   | ×  |     |     |     |

- G1: nonnegative + acyclic + equal weights → D, BF, DAG, BFS.
- G2: negative edges (out: D, BFS) but DAG, no negative cycle → BF, DAG.
- G3: negative edge (out: D, BFS) and a directed cycle (out: DAG), no negative cycle → only BF.

(checked: Python — G1 acyclic/nonneg/all-equal; G2 acyclic with negatives; G3 cyclic with negatives, cycle a→d→c→b→a = +2, Bellman-Ford finds no negative cycle.)

**Problem (b).** Graphs with n vertices, m = n log n edges. Runtimes as a function of n.

**Method.** Substitute m = n log n into each bound.
- Dijkstra (binary heap): O((n+m) log n) = O((n + n log n) log n) = **O(n (log n)²)**.
- Bellman-Ford: O(n·m) = O(n · n log n) = **O(n² log n)**.
- DAG-Shortest-Paths: O(n + m) = **O(n log n)**.

(checked: Python — substituting m = n log n: Dijkstra n(log n)², BF n² log n, DAG n log n.)

## Opgave 8 (10%) — Red-black trees

**Problem (a).** Color the tree to make a valid RB tree (NIL leaves are black squares): root b, children a (left) and d (right); d has children c (left) and e (right); a, c, e are leaves to NIL.

**Type.** Find a valid RB coloring.

**Method.** Enforce: root black; a red node has only black children; every root-to-NIL path has equal black-height (NIL counts black). Push internal nodes to black; flip the deeper-side leaves to red to equalize black-heights.

**Worked solution.** Path b→a→NIL has 1 internal node; path b→d→{c,e}→NIL has 2. Make a, d black and c, e red.
- **Black: b, a, d.  Red: c, e.** Every root-to-NIL path then has black-height 2. (Valid alternative: Black {b, a, c, e}, Red {d}.)

**Problem (b).** Insert 11 (textbook RB-Insert) into: 5(B) root; left 3(B) with red children 2, 4; right 7(B) with left 6(B), right 9(B); 9 has red children 8, 10. Which of T1–T4 results?

**Type.** Trace RB-Insert with fixup.

**Method.** BST-insert as a red leaf, then fix up while parent is red: uncle red → recolor parent+uncle black, grandparent red, move up (Case 1); uncle black → rotate (Cases 2/3). Finally paint root black.

**Worked solution.** Insert 11 right of 5→7→9→10, becomes right child of 10, red. Parent 10 red → violation. Uncle 8 (other child of 9) is red → Case 1: recolor 10, 8 black, 9 red, move z up to 9. z's parent 7 is black → done. Root stays black.
Result: 5(B); left 3(B):2(R),4(R); right 7(B): 6(B), 9(R); 9 → 8(B), 10(B); 10 → right child 11(R).
This is **T2**. (checked: Python — BST-insert puts 11 right of 10; uncle 8 red triggers Case 1, recolor 8/10 black and 9 red, z moves to 9 whose parent 7 is black, loop ends.)

## Opgave 9 (14%) — Loop runtimes in Θ

**Problem.** Give Θ runtime as a function of n.

```
Algoritme1(n)           Algoritme2(n)
  i = n                   i = 1
  while i > 1             j = 1
     j = n                while i <= n
     while j > i             while j <= i
        j = j - 1               j = j + 1
     i = i - 1             i = i * 2

Algoritme3(n)           Algoritme4(n)
  i = 1                   i = 1
  while i <= n            while i <= n
     j = 1                   j = 1
     while j <= i            while j <= i
        j = j + 1               j = j * 2
     i = i * 2            i = i + 1
```

**Type.** Count nested-loop iterations.

**Method.** For each outer-loop i, count inner iterations, then sum. Note whether the inner counter resets each outer pass (yes in 1,3,4; NO in 2). Geometric outer steps (i*=2) give ~log n passes.

**Worked solution.**
- **Alg1.** Outer i from n down to 2; inner counts n down to i = (n−i) steps. Σ_{i=2}^{n}(n−i) ≈ n²/2. **Θ(n²).** (ops/n² → 0.5.)
- **Alg2.** j never resets. Outer i doubles (≈ log n passes); j only ever rises from 1 to the final i (≤ 2n). Total inner increments ≈ n. **Θ(n).** (ops/n → 1.)
- **Alg3.** j resets each pass; inner runs i times for i = 1,2,4,…,≤ n. Sum ≈ 2n (geometric). **Θ(n).** (ops/n → 2.)
- **Alg4.** Outer i = 1..n (n passes); inner doubles j to i → ⌈log₂ i⌉ steps. Σ_{i=1}^{n} log i = log(n!) = Θ(n log n). **Θ(n log n).** (ops/(n log n) → 1.)

(checked: Python at n=2¹⁴ — alg1 ops/n² ≈ 0.50, alg2 ops/n = 1.0, alg3 ops/n ≈ 2.0, alg4 ops/(n log₂ n) ≈ 0.93. Confirms Θ(n²), Θ(n), Θ(n), Θ(n log n).)

## Opgave 10 (15%) — Augmented BST for the bounding rectangle

Set-up: store n points (distinct x-coordinates, used as BST keys). Each node v keeps v.xmax, v.xmin, v.ymax, v.ymin = max/min x and y over v's subtree (subtree includes v). The bounding rectangle of all points is [root.xmin, root.xmax] × [root.ymin, root.ymax].

**(a) Compute a node's four fields in O(1) from its children.** Combine v's own coordinates (v.x, v.y) with the children's stored aggregates:
```
v.xmax = max(v.x, v.left.xmax, v.right.xmax)
v.xmin = min(v.x, v.left.xmin, v.right.xmin)
v.ymax = max(v.y, v.left.ymax, v.right.ymax)
v.ymin = min(v.y, v.left.ymin, v.right.ymin)
```
Max/min over ≤ 3 values → O(1). NIL child: drop its term (treat as −∞ for max, +∞ for min).

**(b) Maintaining the fields under insert/delete in O(log n).** Each field depends only on a node plus its two children (part a). By CLRS Theorem 14.1, an augmentation a node can recompute in O(1) from its own data and its children's fields is maintainable without changing insert/delete asymptotics:
- Insert/delete walk O(log n) nodes; recompute their fields bottom-up, O(1) each → O(log n).
- A rotation changes only the two rotated nodes' subtrees; recompute their fields (bottom node, then top) in O(1). Insert does O(1) rotations, delete O(log n), each O(1) to repair → O(log n) total.

**(c) Area in O(1).** Read the root aggregates: **Area = (root.xmax − root.xmin)·(root.ymax − root.ymin)**, O(1).

**(d) One point on each side, in O(log n).** Sides: x=xmin (left), x=xmax (right), y=ymin (bottom), y=ymax (top); a point is on a side iff it attains that extreme coordinate.
- Left/right: minimum/maximum x = leftmost/rightmost node (x is the key) → follow left/right children from the root. O(log n).
- Bottom/top: minimum/maximum y. Descend from the root guided by subtree y-aggregates — at each node, either stop (if its own y equals the target ymin/ymax) or step into the child whose subtree carries that extreme y. One O(log n) descent.

**(e) Same results with two trees, no augmentation, inserts/deletes still O(log n).** Keep the points in two balanced BSTs: tree X keyed on x, tree Y keyed on y (y-coordinates also distinct).
- Area (c): bounding rectangle = [min x, max x] × [min y, max y]; min/max x = leftmost/rightmost of X, min/max y = leftmost/rightmost of Y. Cache the four extreme pointers and refresh them in O(1) on each insert/delete → Area O(1).
- Four side points (d): leftmost/rightmost of X = left/right-side points; leftmost/rightmost of Y = bottom/top-side points. Each an O(log n) walk down one side (or O(1) from the cached pointers).
- Insert/delete: do it in both trees → O(log n); refresh four cached extremes in O(1).
No per-node aggregate fields needed; the two key orders give x- and y-extremes directly.
