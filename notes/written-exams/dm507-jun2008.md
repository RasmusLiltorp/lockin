## Opgave 1 (15%)

### a (5%) ‚Äî Radix sort, radix 10

**Problem.** Run radix sort with radix 10 on `747, 765, 544, 754, 431, 231, 222`. Show the result after each iteration (pass).

**Type.** Trace LSD radix sort (counting sort per digit).

**Method (any instance).**
1. Radix sort is LSD: sort by least-significant digit first, then next, up to the most-significant. With radix 10, one pass per decimal digit.
2. Each pass is a *stable* distribution into 10 buckets (0..9) by the current digit, then concatenate buckets 0..9 in order. Stability (keep earlier-arrived items first within a bucket) is what makes the whole thing correct.
3. Number of passes = number of digits of the largest number. Here 3 digits ‚Üí 3 passes.

**Worked solution.** Numbers have 3 digits each.

Pass 1, ones digit (7,5,4,4,1,1,2):
```
431, 231, 222, 544, 754, 765, 747
```
Pass 2, tens digit (3,3,2,4,5,6,4):
```
222, 431, 231, 544, 747, 754, 765
```
Pass 3, hundreds digit (2,4,2,5,7,7,7):
```
222, 231, 431, 544, 747, 754, 765
```
Final sorted output: `222, 231, 431, 544, 747, 754, 765`. (checked: Python)

### b (10%) ‚Äî Running-time table

**Problem.** Give best-case, worst-case, and "sorted input" running times for Insertion Sort, Merge Sort, Quicksort. Justify.

**Type.** State/justify known complexity bounds.

| | Best-case | Worst-case | Sorted input |
|---|---|---|---|
| Insertion Sort | $\Theta(n)$ | $\Theta(n^2)$ | $\Theta(n)$ |
| Merge Sort | $\Theta(n\log n)$ | $\Theta(n\log n)$ | $\Theta(n\log n)$ |
| Quicksort | $\Theta(n\log n)$ | $\Theta(n^2)$ | $\Theta(n^2)$ |

**Justification.**
- **Insertion Sort:** best case is an already-sorted array ‚Äî each new element compares once and stops, $\Theta(n)$. Worst case is reverse-sorted: element $i$ shifts back $i$ positions, $\sum i = \Theta(n^2)$. Sorted input is exactly the best case, $\Theta(n)$.
- **Merge Sort:** always splits in half and merges in linear time regardless of input, recurrence $T(n)=2T(n/2)+\Theta(n)=\Theta(n\log n)$ in all cases. Input order does not matter.
- **Quicksort (textbook deterministic pivot = last element):** best case is balanced partitions, $T(n)=2T(n/2)+\Theta(n)=\Theta(n\log n)$. Worst case is maximally unbalanced partitions ($n-1$ vs $0$), $T(n)=T(n-1)+\Theta(n)=\Theta(n^2)$. With a fixed pivot, **already-sorted input is the worst case** ($\Theta(n^2)$), because every partition peels off one element. (Randomized Quicksort would give expected $\Theta(n\log n)$ on sorted input; state which variant you assume.)

## Opgave 2 (10%)

### a (10%) ‚Äî Asymptotic ordering

**Problem.** Give the asymptotic order of: $\sqrt{n},\ 2^n,\ (\log_{10} n)^2,\ n,\ \log_2 n$.

**Type.** Rank functions by asymptotic growth.

**Method.** Group by class: logarithmic $\prec$ poly-logarithmic $\prec$ polynomial (by increasing exponent) $\prec$ exponential. Constant factors (base of log, etc.) do not change the class.

**Worked solution.** From slowest- to fastest-growing:
$$\log_2 n \ \prec\ (\log_{10} n)^2 \ \prec\ \sqrt{n} \ \prec\ n \ \prec\ 2^n$$
Reasons:
- $\log_2 n = \Theta(\log n)$, and $(\log_{10} n)^2 = \Theta((\log n)^2)$ grows faster than $\log n$ but slower than any positive power of $n$ (polylog $\prec$ poly).
- $\sqrt{n}=n^{1/2}\prec n^1=n$ (smaller exponent).
- $2^n$ dominates every polynomial.

So $\log_2 n = o((\log_{10}n)^2) = o(\sqrt n) = o(n) = o(2^n)$. (checked: Python ‚Äî at $n=10^9$: $\log_2 n\approx 30$, $(\log_{10}n)^2=81$, $\sqrt n\approx 31623$, $n=10^9$.)

### d (7%) ‚Äî Which graphs could give the Figure-3 tree under Kruskal + union-by-rank

**Problem.** Figure 3 is a union-by-rank forest tree: root `a` with direct children `b, c, d, e`, and `f` is a child of `e`. Four weighted graphs (Figure 4) are each run through Kruskal. Which could produce this tree? Justify.

Edge data:
- (a) path `a‚Äì2‚Äìb‚Äì4‚Äìc‚Äì6‚Äìd‚Äì8‚Äìe‚Äì10‚Äìf`.
- (b) path `a‚Äì2‚Äìb‚Äì4‚Äìc‚Äì6‚Äìd‚Äì10‚Äìe‚Äì8‚Äìf`.
- (c) hexagon `a‚Äì1‚Äìb‚Äì2‚Äìc‚Äì9‚Äìd‚Äì10‚Äìe‚Äì4‚Äìf‚Äì7‚Äìa`.
- (d) hexagon + 3 diagonals, **all weights 1**.

**Type.** Reason about the disjoint-set forest shape that union-by-rank builds during Kruskal.

**Method.**
1. Kruskal processes edges in increasing weight order, doing `Union` on the endpoints of each non-cycle edge.
2. The *shape* of the resulting union-by-rank tree is determined by the order unions happen and the ranks at each step. The target tree has a root of **rank 2** (it has a child `e` that itself has a child `f`). So a candidate graph must let the union sequence build a rank-2 root with one rank-1 child.
3. A pure "merge singletons into one root in weight order" pattern gives a flat star (depth 1, root rank 1) ‚Äî that cannot match. You need a sub-tree of rank 1 (two nodes joined) that later attaches as a child of the growing root.
4. For all-equal-weight graphs, ties may be broken in any order, so check whether *some* order reproduces the tree.

**Worked solution** (verified by simulation):

- **(a): NO.** Edges added in order 2,4,6,8,10: `a-b` (rank‚Üí1), then `b-c`, `c-d`, `d-e`, `e-f` each merge a singleton into the single rank-1 root. Result is a depth-1 star ‚Äî root with five direct children, no node has a grandchild. Figure 3 has `f` two levels below the root, so (a) is impossible.

- **(b): YES.** Order 2,4,6,8,10 ‚Üí edges sorted: `a-b`(2), `b-c`(4), `c-d`(6), `e-f`(8), `d-e`(10). First three build a rank-2 root over {a,b,c,d} with structure matching (root + direct children + one rank-1 child); `e-f`(8) joins e,f into a rank-1 pair before the linking edge `d-e`(10) attaches that pair under the root. Final shape = root rank 2 with several direct children and one rank-1 child carrying a leaf ‚Äî isomorphic to Figure 3.

- **(c): YES.** Sorted edges: `a-b`(1), `b-c`(2), `e-f`(4), `a-f`(7), `c-d`(9), [`d-e`(10) closes a cycle, skipped]. `e-f`(4) forms a rank-1 pair, which then attaches under the growing root via `a-f`(7); the other nodes hang directly off the root. Same rank-2-root-with-one-rank-1-child shape as Figure 3.

- **(d): YES.** All nine edges have weight 1, so Kruskal may break ties in any order. A valid order (e.g. `a-b, b-c, c-d, e-f, d-e, f-a, ‚Ä¶` with later edges skipped as cycles) builds exactly the Figure-3 shape. Since at least one tie-break order reproduces it, (d) could result in the tree.

**Answer: graphs (b), (c) and (d) could produce the Figure-3 tree; graph (a) cannot.** (checked: Python ‚Äî brute-forced all tie-break orders, edge orientations, and both equal-rank link directions against the exact labeled target tree; (a) is the only graph with no producing run.)

## Opgave 5 (15%) ‚Äî Correctness of binary exponentiation

Algorithm (computes $a^b$ from the bits of $b$, MSB‚ÜíLSB, `B[i]=b_i`, `B[k]` is MSB):
```
Eksp(a, B)
  k ‚Üê length(B) ‚àí 1
  c ‚Üê 1
  for i ‚Üê k downto 0
      c ‚Üê c ¬∑ c
      if B[i] = 1
          c ‚Üê c ¬∑ a
  return c
```
Define $B_{i+1}$ = the number formed by the $k-i$ most significant bits of $b$, so $B_{k+1}=0$, $B_k=b_k$, ‚Ä¶, $B_0=b$.

### a (10%) ‚Äî Prove the loop invariant

**Invariant.** At the *start* of the for-loop body (right after `i` is set), $c = a^{B_{i+1}}$.

**Type.** Loop-invariant proof (initialization / maintenance / termination).

**Method.**
1. **Initialization.** Show the invariant holds before the first iteration ($i=k$).
2. **Maintenance.** Assume it holds at the start of iteration $i$; show that after the body runs and $i$ decreases to $i-1$, it holds again at the start of iteration $i-1$.
3. **Termination.** Use the invariant at loop exit to conclude the algorithm is correct.

**Worked solution.**

Key relation between consecutive $B$ values. By definition,
$$B_i = 2\,B_{i+1} + b_i.$$
(Shifting the prefix left one bit and appending bit $b_i$.) This is what one body iteration must realize, because $a^{B_i} = a^{2B_{i+1}+b_i} = (a^{B_{i+1}})^2 \cdot a^{b_i}$.

**Initialization** ($i=k$, first entry). Before the body, $c=1$. The invariant claims $c=a^{B_{k+1}}=a^0=1$. True.

**Maintenance.** Assume at the start of iteration $i$ that $c=a^{B_{i+1}}$. The body does:
- `c ‚Üê c¬∑c`: now $c = (a^{B_{i+1}})^2 = a^{2B_{i+1}}$.
- `if B[i]=1: c ‚Üê c¬∑a`: this multiplies by $a^{b_i}$ (by $a$ when $b_i=1$, by nothing when $b_i=0$). Now
$$c = a^{2B_{i+1}} \cdot a^{b_i} = a^{2B_{i+1}+b_i} = a^{B_i}.$$
After the body, the loop counter becomes $i-1$, and at the start of that next iteration the invariant claims $c=a^{B_{(i-1)+1}}=a^{B_i}$. That is exactly what we have. Maintenance holds.

**Termination.** The loop ends after the iteration with $i=0$. By maintenance, after that iteration $c=a^{B_0}$. Since $B_0=b$, we have $c=a^b$. (Conceptually the invariant "at start of iteration $i$" is $c=a^{B_{i+1}}$; pushing it one past the last iteration gives $c=a^{B_0}$.)

### b (5%) ‚Äî Conclude correctness

**Type.** Use the invariant to argue total correctness.

**Worked solution.** The loop runs $i=k,k-1,\dots,0$, a finite count of $k+1$ iterations, so it terminates. By the termination step of part (a), at exit $c=a^{B_0}=a^b$. The algorithm returns this $c$, so `Eksp(a,B)` returns $a^b$. Correct. (Running time: $\Theta(k)=\Theta(\log b)$ multiplications.)

## Opgave 6 (25%) ‚Äî DP path cover of a weighted binary tree

A **path cover** (sti-d√¶kning) is a subset $X$ of nodes such that every root-to-leaf path contains *exactly one* node of $X$. Maximize total weight. $W(x)$ = max weight of an optimal path cover of the subtree rooted at $x$.

Figure 6 tree (weights):
```
            a:12
          /      \
       b:8        c:10
      /   \       /   \
   d:2    e:2  f:5    g:7
                      /   \
                   h:6    i:4
```

### a (10%) ‚Äî Fill the W table

**Type.** Bottom-up DP on a tree.

**Method.** For each node $x$ with weight $w(x)$:
- If $x$ is a **leaf**, the only path through it is the path ending at $x$; it must be the chosen node ‚áí $W(x)=w(x)$.
- If $x$ is **internal** with children $c_1,\dots$: either pick $x$ itself (covers every path through $x$ with one node, value $w(x)$), **or** pick nothing at $x$ and solve each child subtree independently (every root-to-leaf path goes through exactly one child, so the choices are independent and add):
$$W(x)=\max\Big(w(x),\ \sum_{c \text{ child of } x} W(c)\Big).$$
Compute leaves first, then work up to the root.

**Worked solution.**
- Leaves: $W(d)=2,\ W(e)=2,\ W(f)=5,\ W(h)=6,\ W(i)=4$.
- $W(g)=\max(7,\ W(h)+W(i))=\max(7,10)=10$.
- $W(b)=\max(8,\ W(d)+W(e))=\max(8,4)=8$.
- $W(c)=\max(10,\ W(f)+W(g))=\max(10,15)=15$.
- $W(a)=\max(12,\ W(b)+W(c))=\max(12,23)=23$.

| x | a | b | c | d | e | f | g | h | i |
|---|---|---|---|---|---|---|---|---|---|
| W(x) | **23** | 8 | 15 | 2 | 2 | 5 | 10 | 6 | 4 |

(checked: Python ‚Äî bottom-up DP gives W = {a:23, b:8, c:15, d:2, e:2, f:5, g:10, h:6, i:4}.)

### b (5%) ‚Äî An optimal path cover

**Type.** Reconstruct the chosen node set.

**Method.** Top-down: at each node, if $W(x)=w(x)$ and that beat the children sum, select $x$ and stop descending; otherwise recurse into the children.

**Worked solution.** At `a`: children sum $8+15=23 > 12$, so don't pick `a`; recurse.
- At `b`: $w=8 \ge W(d)+W(e)=4$, so pick **b**, stop.
- At `c`: children sum $W(f)+W(g)=5+10=15 > 10$, recurse.
  - At `f` (leaf): pick **f**.
  - At `g`: children sum $W(h)+W(i)=6+4=10 > 7$, recurse ‚Üí pick **h** and **i**.

Optimal cover: $X=\{b, f, h, i\}$, total weight $8+5+6+4 = 23$.
Check every root-to-leaf path hits exactly one: a-b-d ‚úì(b), a-b-e ‚úì(b), a-c-f ‚úì(f), a-c-g-h ‚úì(h), a-c-g-i ‚úì(i). Good.

### c (10%) ‚Äî DP algorithm, time and space

**Type.** State the algorithm and analyze cost.

**Method / algorithm.** Compute $W$ by a post-order traversal of the tree (children before parent), applying the recurrence above:
```
PathCover(x):
    if x is a leaf:
        W[x] ‚Üê weight(x)
    else:
        s ‚Üê 0
        for each child c of x:
            PathCover(c)        // recurse first
            s ‚Üê s + W[c]
        W[x] ‚Üê max(weight(x), s)
    return W[x]
```
Call `PathCover(root)`; the answer is `W[root]`. To also output the set, do the top-down reconstruction from part (b) (or store a boolean "picked here" at each node).

**Correctness.** Optimal substructure: a path cover restricted to a subtree rooted at $x$ is itself a path cover of that subtree, and for an internal $x$ either $x\in X$ (then no descendant of $x$ may be in $X$, value $w(x)$) or $x\notin X$ (then each child subtree is covered independently and the values add). Taking the max gives $W(x)$. Induction up the tree gives the global optimum at the root.

**Running time.** Each node is visited once; work per node is $O(\#\text{children})=O(1)$ for a binary tree (constant work plus summing two children). Total $\Theta(n)$ for $n$ nodes.

**Space.** $\Theta(n)$ for the $W$ table (one value per node), plus recursion stack $O(h)$ where $h$ is the tree height ($O(n)$ worst case). So $\Theta(n)$ space.
