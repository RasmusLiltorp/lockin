### 1b. Insert key 5 into the given RB tree; show every step.

**Worked solution.**

## Problem 2 (Opgave 2, 15%) ŌĆö Mult correctness and recurrence

The algorithm (positive integers `a`, `b`):
```
Mult(a, b)
  if a == 1: return b
  else: return Mult( floor(a/2), 2b ) + b*(a mod 2)
```

### 2a. Prove by induction on `a` that `Mult(a,b)` returns `ab`.

**Type.** Correctness proof by strong induction on the recursion argument.

**Method (any such recursion).** Pick the induction variable (here `a`). Prove the base case directly. Assume the claim for all smaller values (strong/complete induction), substitute the recursive call's value by the hypothesis, and simplify to the target.

**Worked solution.**

Claim: for all integers `a Ōēź 1`, `b Ōēź 1`, `Mult(a,b) = a┬Ęb`.

Strong induction on `a`.

*Base* `a = 1`: returns `b = 1┬Ęb`. Ō£ō

*Step* `a Ōēź 2`: assume `Mult(a', b') = a'┬Ęb'` for all `a' < a` and all `b'`. The call returns
`Mult(ŌīŖa/2Ōīŗ, 2b) + b┬Ę(a mod 2)`.
Since `a Ōēź 2`, `ŌīŖa/2Ōīŗ < a`, so by the hypothesis `Mult(ŌīŖa/2Ōīŗ, 2b) = ŌīŖa/2Ōīŗ┬Ę2b`. Hence the return value is
`2b┬ĘŌīŖa/2Ōīŗ + b┬Ę(a mod 2) = b┬Ę(2ŌīŖa/2Ōīŗ + (a mod 2)) = b┬Ęa`,
using `2ŌīŖa/2Ōīŗ + (a mod 2) = a` (the integer-division identity). Ō£ō

By induction `Mult(a,b) = ab` for all `a Ōēź 1`. (Checked numerically for all `aŌłł[1,200], bŌłł[1,50]`.)

### 2b. Recurrence for the running time `T(a)` and its solution.

**Type.** Set up and solve a divide-by-constant recurrence.

**Method.** Count the work outside the recursive call (here `O(1)`: one floor, one multiply by 2, one mod, one add). The argument shrinks from `a` to `ŌīŖa/2Ōīŗ`. Write `T(a) = T(ŌīŖa/2Ōīŗ) + O(1)`, then solve (Master theorem or by unrolling).

**Worked solution.**

`T(1) = ╬ś(1)`, and for `a Ōēź 2`: `T(a) = T(ŌīŖa/2Ōīŗ) + ╬ś(1)`.

The argument halves each level, so the recursion depth is about `logŌéé a`, with `O(1)` work per level. Master theorem with `a_M=1, b_M=2, f(n)=╬ś(1)`: `n^{log_b a}=n^0=1=f`, case 2, giving `╬ś(log a)`.

**`T(a) = ╬ś(log a)`.**

## Problem 4 (Opgave 4, 25%) ŌĆö block-score DP

For strings `S`, `T`, a common subsequence `F` is split into the minimum number `b` of blocks, each block a *contiguous* substring of both `S` and `T`. The **block-score** (blokscore) of `F` is `|F| ŌłÆ b`. The block-score of the pair is the max over all common subsequences. `B(i,j)` = block-score of prefixes `SßĄó = sŌéüŌĆ”sßĄó` and `TŌ▒╝ = tŌéüŌĆ”tŌ▒╝`.

(Worked example from the sheet: `S = ab┬Ęcc┬Ębaca┬Ęba┬Ęabad`, `T = ab┬Ęaba┬Ębaca┬Ęabad┬Ęad`, blocks `ab|baca|abad` giving `2+4+4ŌłÆ3 = 7`. Verified in code.)

### 4a. For `S = bbcba`, `T = bccbba`, fill the table with `B(i,j)`.

**Type.** Hand-fill a DP table from a recurrence.

**Method.** Initialize row 0 and column 0 to 0. Fill left-to-right, top-to-bottom using the recurrence (4b). On a match, also compute `m_ij` = length of the longest common suffix of `SßĄó` and `TŌ▒╝`, and consider `B(iŌłÆm_ij, jŌłÆm_ij) + m_ij ŌłÆ 1`.

**Worked solution** (rows `i`=prefix of `S`, cols `j`=prefix of `T`):

```
          j: 0  1  2  3  4  5  6
                b  c  c  b  b  a
   i=0       0  0  0  0  0  0  0
   i=1  b    0  0  0  0  0  0  0
   i=2  b    0  0  0  0  0  1  1
   i=3  c    0  0  1  1  1  1  1
   i=4  b    0  0  1  1  1  1  1
   i=5  a    0  0  1  1  1  1  2
```

**Final block-score `B(5,6) = 2`.** (checked: Python ŌĆö whole table reproduced.)

Where the 2 comes from: the best common subsequence is `F = "bbba"`, split into 2 blocks `"bb" | "ba"`. Both blocks are contiguous in `S=bbcba` ("bb" at the front, "ba" at the end) and in `T=bccbba` (both inside the trailing "bba"). Score `= |F| ŌłÆ b = 4 ŌłÆ 2 = 2`. (checked: Python ŌĆö brute-forced over all common subsequences, max block-score is 2 with witness "bbba".)

### 4b. Justify the recurrence.

Recurrence (given):
```
B(i,j) = 0                                                  if i=0 or j=0
B(i,j) = max( B(i,j-1), B(i-1,j) )                          if s_i ŌēĀ t_j
B(i,j) = max( B(i,j-1), B(i-1,j),
              B(i-m_ij, j-m_ij) + m_ij - 1 )                if s_i = t_j
m_ij = max{ k : s_{i-Ōäō}=t_{j-Ōäō} for 0ŌēżŌäōŌēżk-1 }  (longest common suffix length)
```

**Justification.** Look at the optimal common subsequence `F` of `SßĄó, TŌ▒╝`.

- **Base:** an empty prefix has no common subsequence, score 0.
- **If `sßĄó ŌēĀ tŌ▒╝`:** `sßĄó` and `tŌ▒╝` cannot both be the last matched char in the same block-aligned way; the optimum drops one of them, so it is the best of dropping the last char of `T` (`B(i,j-1)`) or of `S` (`B(i-1,j)`). Adding a non-matching char never helps, so no third term.
- **If `sßĄó = tŌ▒╝`:** either the optimum still ignores this matched pair (first two terms, same as above), **or** it ends with a block that lines up the matching suffix. The longest suffix that matches in both prefixes has length `m_ij`. Using a block of length `m_ij` adds `m_ij` characters but costs `1` block (the `ŌłÆb` in the score), net `+m_ij ŌłÆ 1`, and the rest is the best score on the prefixes before that block: `B(iŌłÆm_ij, jŌłÆm_ij)`. Take the max of the three.

The `max` of the three options covers every case, so `B` is correct.

### 4c. Pseudocode (dynamic programming) and running time.

**Type.** Turn the recurrence into a bottom-up algorithm.

```
BlockScore(S[1..m], T[1..n]):
  for i = 0..m: B[i][0] = 0
  for j = 0..n: B[0][j] = 0
  for i = 1..m:
    for j = 1..n:
      if S[i] != T[j]:
        B[i][j] = max(B[i][j-1], B[i-1][j])
      else:
        # longest common suffix length m_ij
        k = 0
        while i-k >= 1 and j-k >= 1 and S[i-k] == T[j-k]:
          k = k + 1
        mij = k
        opt = B[i-mij][j-mij] + mij - 1
        B[i][j] = max(B[i][j-1], B[i-1][j], opt)
  return B[m][n]
```

**Running time.** `╬ś(mn)` cells; each match cell may scan back up to `min(i,j)` chars to find `m_ij`, costing `O(min(m,n))`. Worst case **`O(m┬Ęn┬Ęmin(m,n))`** (e.g. `O(n┬│)` for equal lengths). If `m_ij` is precomputed for all `(i,j)` with a longest-common-suffix table (another `╬ś(mn)` DP: `L[i][j]=L[i-1][j-1]+1` when `s_i=t_j`, else 0), each cell is `O(1)` and the whole thing is **`╬ś(mn)`**. State the bound you implement; the simple version above is `O(mn┬Ęmin(m,n))`.

## Problem 5 (Opgave 5, 20%) ŌĆö sort n integers with few distinct values

### 5a. Sort `n` integers in `O(n log log n)` when there are only `O(log n)` distinct values.

**Type.** Algorithm design + analysis; exploit few distinct keys. Comparisons are `O(1)`; key sizes may be huge (so radix/counting on the value range is not allowed).

**Method (general recipe).** Keep a search structure keyed on *distinct* values only, where each entry carries a multiplicity counter. Insert each of the `n` items: search for its value; if present, increment the counter; else insert a new entry. Then read the structure in sorted order, emitting each value as many times as its count.

**Worked solution.**

Use a **balanced binary search tree** (r├Ėd-sort tr├” / 2-3 tree ŌĆö anything with `O(log size)` ops) whose nodes are *(value, count)*, one node per distinct value.

Processing:
```
for each of the n input numbers x:
    search the BST for x
    if found: node.count += 1
    else:     insert (x, 1)
```
Let `d` be the number of distinct values, given `d = O(log n)`. The tree never holds more than `d` nodes, so each search/insert costs `O(log d)`. With `d = O(log n)`:
`log d = log(O(log n)) = O(log log n)`.
Total over `n` items: `n ┬Ę O(log log n) = O(n log log n)`. Ō£ō

The numbers can be astronomically large; we only ever **compare** them (`O(1)` each), never index by value, so the bound holds regardless of magnitude. (The hint ŌĆö "use a suitable data structure to insert into, and collect the duplicates" ŌĆö is exactly this.)

### 5b. How to read out the sorted sequence and print it in `O(n)`.

**Type.** Output extraction from the structure.

**Method / worked solution.** Do an **in-order traversal** (inorden-genneml├Ėb) of the BST. In-order visits values in increasing order. At each node *(value, count)*, print `value` exactly `count` times.

Cost: the traversal touches the `d` nodes in `O(d)` total, and prints `╬Ż count = n` numbers, one output op each. Since `d = O(log n) = O(n)`, total output time is `O(d + n) = O(n)`. Ō£ō
