## Opgave 1 (20%) — heap, red-black tree, hashing

### 1a (7%) Heap-Extract-Max

**Problem.** Run `Heap-Extract-Max` on the binary max-heap stored as the array
`[10, 7, 6, 5, 4, 2, 3, 1, 2, 3, 1, 1]` (1-indexed). Show each step; drawing the tree is recommended.

**Type.** Trace a heap operation (`Heap-Extract-Max` = return root, move last element to root, shrink, `Max-Heapify(1)`).

**Method (any instance).**
1. Save `max = A[1]` (the root). This is the returned value.
2. Move the last element `A[heap-size]` into `A[1]`. Decrement `heap-size` by 1.
3. Run `Max-Heapify` from the root: compare node with its two children, swap with the larger child if a child is larger, follow the element down, repeat until the heap property holds.
4. The array after the sift-down is the new heap.

**Worked solution.**
Array indices 1..12: `10 7 6 5 4 2 3 1 2 3 1 1`. As a tree:
```
            10
        7        6
      5    4    2   3
    1  2  3 1  1
```
- Step 1: `max = 10`.
- Step 2: move last element `1` (index 12) to the root, heap-size becomes 11.
  Root area now: `1 7 6 5 4 2 3 1 2 3 1`.
- Step 3 `Max-Heapify(1)`:
  - root `1` vs children `7` (idx2) and `6` (idx3): largest is `7`. Swap idx1↔idx2.
  - `1` at idx2 vs children `5` (idx4), `4` (idx5): largest `5`. Swap idx2↔idx4.
  - `1` at idx4 vs children `1` (idx8), `2` (idx9): largest `2`. Swap idx4↔idx9.
  - idx9 has no children. Stop.

**Final heap array (size 11):** `[7, 5, 6, 2, 4, 2, 3, 1, 1, 3, 1]`. (checked: Python)
As a tree:
```
            7
        5        6
      2    4    2   3
    1  1  3 1
```
Returned value: **10**.

### 1b (6%) Red-black tree deletion

**Problem.** Delete the node with key 2 from the red-black tree below (black nodes drawn bold). Draw the resulting tree.

Input tree (B = black/bold, R = red):
```
                9(B)
          2(R)         12(B)
       1(B)  7(B)         15(R)
            5(R) 8(R)
```
(`12`'s left child is NIL; `1`'s and `15`'s children are NIL.)

**Type.** Trace `RB-Delete` (CLRS): node with two children → replace by successor, then fix up if a black node was removed.

**Method (any instance).**
1. If the node `z` has two non-NIL children, find `y =` successor `= minimum of z`'s right subtree. The successor's original color `y-original-color` is what matters for fixup. `x` is `y`'s right child (the node that moves into `y`'s old spot).
2. Splice `y` out, put it where `z` was, and give `y` the color `z` had. (One child / leaf cases are simpler: splice directly.)
3. If `y-original-color` was BLACK, a black node left the tree → run `RB-Delete-Fixup(x)` to restore the black-height property (recoloring + rotations). If it was RED, no fixup needed.

**Worked solution.**
- `z =` node 2, two children → successor `y = minimum(right subtree of 2) = minimum(subtree at 7) =` node **5**. `5` is RED, `x = 5.right = NIL`.
- Move `5` up into 2's position and recolor `5` to 2's color (RED). `7` keeps its black color; `7`'s left child becomes NIL (5 left there), right stays 8(R). `1(B)` becomes 5's left child.
- `y-original-color = RED` → **no fixup needed**. Tree stays valid.

**Resulting tree:**
```
                9(B)
          5(R)         12(B)
       1(B)  7(B)         15(R)
                8(R)
```
(5 is red, with black children 1 and 7; 7 has only the red right child 8; 12 has only the red right child 15.) Black-heights are consistent and no red node has a red child. (checked: Python — CLRS RB-Delete simulation gives exactly this tree, y-original-color RED so no fixup.)

### 1c (7%) Hashing — quadratic probing

**Problem.** Hash table, 16 slots, open addressing, quadratic probing with `h0(k) = k mod 16`, `c1 = c2 = 1/2`. Occupied slots: 0→32, 4→20, 7→23, 8→7, 9→39. Insert key 71. Which slot does it land in?

**Type.** Trace quadratic probing: `h(k,i) = (h0(k) + c1·i + c2·i²) mod m`, `i = 0,1,2,...`

**Method (any instance).**
1. Compute base `h0(k) = k mod m`.
2. For `i = 0,1,2,...` compute `h(k,i) = (h0(k) + 0.5·i + 0.5·i²) mod 16`.
3. First empty slot encountered is the landing slot.

**Worked solution.** `h0(71) = 71 mod 16 = 7`.
- `i=0`: `(7 + 0 + 0) mod 16 = 7` → occupied (23).
- `i=1`: `(7 + 0.5 + 0.5) mod 16 = 8` → occupied (7).
- `i=2`: `(7 + 1 + 2) mod 16 = 10` → empty.

**Key 71 lands in slot 10.** (checked: Python — probe sequence 7, 8, 10.) (Note: with `c1=c2=1/2`, `0.5i+0.5i² = i(i+1)/2`, which gives offsets 0,1,3,6,10,... — the triangular numbers.)

## Opgave 2 (15%) — loop-invariant correctness

```
KvadratRod(n)
  i ← 0
  s ← 0
  while s ≤ n
     s ← s + 2i + 1
     i ← i + 1
  r ← i − 1
  return r
```

### 2a (8%) Prove the invariant

**Problem.** Prove the while-loop invariant: *at the start of the while-loop (i.e. at the loop test), `s = i²`.*

**Type.** Loop-invariant proof (initialization / maintenance / termination), CLRS style.

**Method (any instance).**
1. **Initialization:** show the invariant holds the first time the test is reached.
2. **Maintenance:** assume it holds at the start of an iteration; show one iteration's body re-establishes it for the next test.
3. **Termination:** combine the invariant with the loop-exit condition to conclude something about the output.

**Worked solution.**
- **Initialization.** Before the first test, `i = 0` and `s = 0`. So `s = 0 = 0² = i²`. Holds.
- **Maintenance.** Assume at the start of an iteration `s = i²`. The body sets
  `s' = s + 2i + 1` and `i' = i + 1`. Then
  `s' = i² + 2i + 1 = (i+1)² = (i')²`.
  So at the next test `s = i²` again holds.
- By induction the invariant holds at every loop test. ∎

### 2b (7%) Use the invariant to prove the return value

**Problem.** Use the invariant to argue `KvadratRod` returns the largest integer ≤ √n, i.e. `r = ⌊√n⌋`.

**Type.** Correctness argument from invariant + termination condition.

**Method.** At loop exit the test `s ≤ n` is false, so `s > n`. Combine with the invariant value of `s` and the relation between `i` and the previous iteration to pin down `r`.

**Worked solution.**
The loop exits the first time `s > n`. Let `i` be the value of `i` at that moment (right after the test fails — the invariant `s = i²` holds at the test). So:
- `i² = s > n`, hence `i > √n`, i.e. `i ≥ ⌊√n⌋ + 1`.
- The previous iteration's test passed, meaning before the last body execution `s_prev = (i-1)² ≤ n`, hence `(i-1) ≤ √n`, i.e. `i - 1 ≤ √n`.

So `i - 1 ≤ √n < i`. Therefore `i - 1 = ⌊√n⌋` (the unique integer `≤ √n` whose successor is `> √n`). The algorithm returns `r = i - 1 = ⌊√n⌋`, the largest integer ≤ √n. ∎
(Checked numerically: returns `isqrt(n)` for n = 0..100.)

## Opgave 3 (20%) — choose simplest shortest-path algorithm + distances

**Problem.** For each of the three directed graphs `G1, G2, G3`, give the *simplest possible* algorithm to find shortest paths from `s` to every other vertex, justify the choice, and give all distances. (All three distance tables checked: Python — Bellman-Ford for G1/G2, Dijkstra for G3.)

**Type.** Algorithm selection by edge-weight / structure, then compute the distances.

**Method (decision rule — the thing to plug in for any graph).**
1. **All edge weights equal (or unweighted)?** → use **BFS** (`O(V+E)`); each edge costs the same so BFS layers = shortest distances.
2. **Has negative weights but is a DAG (no cycles)?** → relax edges in **topological order** (`O(V+E)`). Negative weights are fine, no cycles means one pass suffices. (Dijkstra would be *wrong* with negatives.)
3. **Non-negative weights, may contain cycles?** → **Dijkstra** (`O((V+E) log V)` with a binary heap).
4. (Negative weights *and* cycles → Bellman-Ford; not needed here.)

I label the 3×3 grid `v_rc`, row `r=0..2` top→bottom, col `c=0..2` left→right; `s = v00`.

### G1 — all weights 1 (directed)

Edges (all weight 1): `s→01`, `02→01`, `10→00`, `10→20`, `10→21`, `01→11`, `01→12`, `12→02`, `12→22`, `11→10`, `11→12`, `21→11`, `20→21`, `22→21`.

**Algorithm:** **BFS** from `s`. All edges have the same weight 1, so BFS distance (number of edges) × 1 = shortest path. Simplest and `O(V+E)`.

**Distances from s:**

| v00 | v01 | v02 | v10 | v11 | v12 | v20 | v21 | v22 |
|----|----|----|----|----|----|----|----|----|
| 0  | 1  | 3  | 3  | 2  | 2  | 4  | 4  | 3  |

### G2 — negative weights, but a DAG (directed)

Edges: `s→01 (3)`, `01→02 (-5)`, `s→10 (1)`, `01→11 (-5)`, `01→12 (2)`, `02→12 (4)`, `10→11 (1)`, `11→12 (1)`, `10→20 (4)`, `10→21 (-2)`, `11→21 (2)`, `12→22 (2)`, `20→21 (1)`, `21→22 (3)`. Every edge points right, down, or diagonally down-right → acyclic.

**Algorithm:** relax edges in **topological order** (`DAG-Shortest-Paths`, `O(V+E)`). It is a DAG so a single topological pass is enough, and unlike Dijkstra it handles the negative weights (-5, -2) correctly. A valid topo order: `v00, v01, v02, v10, v11, v12, v20, v21, v22`.

**Distances from s:**

| v00 | v01 | v02 | v10 | v11 | v12 | v20 | v21 | v22 |
|----|----|-----|----|-----|-----|----|-----|----|
| 0  | 3  | -2  | 1  | -2  | -1  | 5  | -1  | 1  |

(E.g. `v02 = 3 + (-5) = -2`; `v11 = min(3-5, 1+1) = -2`; `v21 = min(1-2, -2+2) = -1`.)

### G3 — non-negative weights, has cycles (directed)

Edges: `s→01 (1)`, `01→02 (3)`, `s→10 (2)`, `01→11 (2)`, `12→01 (1)`, `02→12 (4)`, `11→10 (5)`, `11→12 (5)`, `10→20 (2)`, `21→10 (1)`, `21→11 (2)`, `12→22 (4)`, `20→21 (1)`, `21→22 (2)`. There are cycles (e.g. `01→11→12→01`).

**Algorithm:** **Dijkstra** (`O((V+E) log V)`). All weights are non-negative so Dijkstra is correct; BFS won't work because weights differ, and topo-order relaxation won't work because the graph has cycles.

**Distances from s:**

| v00 | v01 | v02 | v10 | v11 | v12 | v20 | v21 | v22 |
|----|----|----|----|-----|-----|----|-----|----|
| 0  | 1  | 4  | 2  | 3   | 8   | 4  | 5   | 7  |

(E.g. `v10 = 2`; `v11 = 1+2 = 3`; `v20 = 2+2 = 4`; `v21 = 4+1 = 5`; `v22 = 5+2 = 7`; `v12 = 4+4 = 8`.)

## Opgave 4 (20%) — minimum vertex cover of a tree (greedy)

A vertex cover (knude-dækning) of `G=(V,E)` is `V' ⊆ V` such that every edge is incident to at least one vertex in `V'`. We want a minimum vertex cover of a tree. `e(v)` = set of edges incident to `v`.

```
Knude-Dækning(V, E)
  V' ← ∅
  while E ≠ ∅
     choose a vertex v that has a child u with no (remaining) children   # u is a leaf
     V' ← V' ∪ {v}
     E ← E − e(v)
  Return V'
```
(Pick a deepest internal node `v` — the parent of a current leaf — add `v`, delete all edges touching `v`.)

### 4a (10%) Prove the greedy finds a minimum vertex cover

**Type.** Greedy correctness via an **exchange argument** (there is an optimal solution that agrees with the greedy choice).

**Method (any greedy-on-trees proof).**
1. **Coverage / termination:** the loop removes edges until `E=∅`, and every removed edge was incident to a vertex put in `V'`, so `V'` is a vertex cover.
2. **Exchange / greedy-choice property:** show some optimal cover contains the greedy choice. Take a leaf `u` with parent `v`. Edge `(u,v)` must be covered, so any cover contains `u` or `v`. Argue `v` is at least as good as `u`.
3. **Optimal substructure / induction:** after committing the greedy choice and deleting covered edges, the rest is a smaller instance; induction finishes it.

**Worked solution.**
1. **`V'` is a cover.** Each iteration deletes `e(v)`, all edges incident to the chosen `v`, and these are exactly the edges `v` covers. The loop ends only when `E=∅`, so every edge got deleted by being incident to some chosen vertex. Hence every edge is covered by `V'`.

2. **Greedy choice is in some optimum.** Let `u` be a leaf in the current tree and `v` its parent (the algorithm always picks such a `v`). The edge `(u,v)` must be covered, so any vertex cover `C` contains `u` or `v`.
   - If `v ∈ C`: fine, `C` already agrees with the greedy choice.
   - If `v ∉ C` then `u ∈ C`. Build `C' = C − {u} + {v}`. `|C'| ≤ |C|`. `C'` is still a cover: the only edge `u` covered is `(u,v)` (u is a leaf, degree 1), and `v` covers that edge too; every other edge `u` did not touch. So `C'` covers everything `C` did, and `|C'| ≤ |C|`. So there is a minimum cover containing `v`.
   Choosing `v` over `u` never hurts because `v` can cover `(u,v)` *and* the edge from `v` to its own parent, while `u` covers only `(u,v)`.

3. **Induction.** After adding `v` and deleting `e(v)`, the remaining edges form a smaller forest. By induction the algorithm finds a minimum cover of the remainder. Since there is a minimum cover of the whole tree that contains `v` (step 2), and the rest is covered minimally, the union `{v} ∪ (greedy on remainder)` is a minimum vertex cover of the whole tree. ∎

### 4b (10%) Linear-time implementation

**Type.** Implement the greedy in `O(n)`.

**Method / answer.**
Root the tree anywhere and process vertices **bottom-up** (children before parents), e.g. via a post-order DFS or by processing vertices in decreasing depth order. Keep a boolean `inCover[v]` (initially false) and, for each vertex, whether it still has an uncovered edge to its parent.

Post-order traversal:
```
DFS(v):
  for each child c of v: DFS(c)
  if some child c of v has NOT yet been covered by an edge      # i.e. edge (v,c) still uncovered
       and that edge is uncovered:
     add v to V'                    # v covers (v,c) and the edge (v,parent)
```
Concretely, process in post-order; when visiting `v`, if any edge from `v` to a child is still uncovered, put `v` in the cover and mark all edges incident to `v` (to children and to its parent) as covered.

Each vertex is visited once, and each edge is examined a constant number of times (once from each endpoint). The DFS is `O(V+E)`; a tree has `E = n-1`, so the total is **`O(n)`**.

## Opgave 5 (25%) — dynamic programming: longest palindromic subsequence

`S = x1…xn`. Find the length of a longest subsequence of `S` that is a palindrome. Recursive spec (1-indexed):
```
LP(S, i, j)
  if i > j:  return 0
  if i = j:  return 1
  if i < j and x_i = x_j:  return 2 + LP(i+1, j-1)
  if i < j and x_i ≠ x_j:  return max{ LP(i+1, j), LP(i, j-1) }
```
(The exam's printout writes `i=1` in the second base case; that is a typo for `i=j`.) Call `LP(S,1,n)`.

### 5a (9%) Fill the table for S = ALBA

**Type.** Fill a 2-D DP table from a recurrence.

**Method.** Index `i` = row (start), `j` = column (end). Only the upper triangle `i ≤ j` is used. Base: diagonal `LP(i,i)=1`, and `i>j` ⇒ 0. Fill by increasing substring length `j−i`.

**Worked solution.** `S = A L B A` (`x1=A, x2=L, x3=B, x4=A`).

|       | j=1 | j=2 | j=3 | j=4 |
|-------|-----|-----|-----|-----|
| i=1   | 1   | 1   | 1   | **3** |
| i=2   | 0   | 1   | 1   | 1   |
| i=3   | 0   | 0   | 1   | 1   |
| i=4   | 0   | 0   | 0   | 1   |

`LP(1,4)`: `x1 = x4 = A` → `2 + LP(2,3)`. `LP(2,3)`: `L≠B` → `max(LP(3,3), LP(2,2)) = 1`. So `LP(1,4) = 2 + 1 = 3`. **Answer: 3** (the palindrome `ABA` or `ALA`). (checked: Python — full table matches.)

### 5b (11%) Iterative DP algorithm

**Type.** Convert the recurrence to a bottom-up table fill. Fill order = increasing substring length so every sub-call is already computed.

```
LP-Iter(x[1..n]):
  let L[1..n][1..n] be a new table
  for i = 1 to n:
     L[i][i] = 1                       # single character
  for len = 2 to n:                    # substring length
     for i = 1 to n - len + 1:
        j = i + len - 1
        if x[i] == x[j]:
           if len == 2: L[i][j] = 2
           else:        L[i][j] = 2 + L[i+1][j-1]
        else:
           L[i][j] = max(L[i+1][j], L[i][j-1])
  return L[1][n]
```
Filling by increasing `len` guarantees `L[i+1][j-1]`, `L[i+1][j]`, `L[i][j-1]` are all already set. (The `len==2` guard handles the `i+1 > j-1` empty range, which contributes 0.)

### 5c (5%) Running time and space

**Type.** Complexity of the table DP.

There are `O(n²)` table entries, each computed in `O(1)`.
- **Running time:** `Θ(n²)`.
- **Space:** `Θ(n²)` for the table. (Reducible to `Θ(n)` if only the length is needed, by keeping two rows; but the straightforward algorithm uses `Θ(n²)`.)
