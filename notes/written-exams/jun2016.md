## Problem 1 (10%) — Recurrences and the Master theorem

**Problem.** For each recurrence: (a) say whether the textbook Master theorem applies and which case; (b) give the solution $T(n)$ where it applies.
- i) $T(n)=2T(n/2)+1$
- ii) $T(n)=3T(n/3)+n$
- iii) $T(n)=4T(n/4)+n\log n$
- iv) $T(n)=5T(n/5)+n^2$

**Type.** Solve divide-and-conquer recurrences with the Master theorem.

**Method (plug in your own $a,b,f(n)$).**
1. Write the recurrence as $T(n)=aT(n/b)+f(n)$. Read off $a$, $b$, $f(n)$.
2. Compute the watershed exponent $\log_b a$ and the comparison function $n^{\log_b a}$.
3. Compare $f(n)$ to $n^{\log_b a}$:
   - **Case 1**: $f(n)=O(n^{\log_b a-\epsilon})$ for some $\epsilon>0$ (f polynomially smaller) $\Rightarrow T(n)=\Theta(n^{\log_b a})$.
   - **Case 2**: $f(n)=\Theta(n^{\log_b a})$ $\Rightarrow T(n)=\Theta(n^{\log_b a}\log n)$.
   - **Case 3**: $f(n)=\Omega(n^{\log_b a+\epsilon})$ for some $\epsilon>0$ AND regularity $af(n/b)\le c f(n)$, $c<1$ $\Rightarrow T(n)=\Theta(f(n))$.
4. If $f(n)$ sits between the cases (only by a non-polynomial factor like $\log n$), the textbook Master theorem does NOT apply.

**Worked solution.**

i) $a=2,b=2$, $\log_b a=\log_2 2=1$, so compare with $n^1=n$. $f(n)=1=O(n^{1-\epsilon})$ (take $\epsilon=1$). **Case 1**. $T(n)=\Theta(n^{\log_b a})=\Theta(n)$.

ii) $a=3,b=3$, $\log_3 3=1$, compare with $n$. $f(n)=n=\Theta(n)$. **Case 2**. $T(n)=\Theta(n\log n)$.

iii) $a=4,b=4$, $\log_4 4=1$, compare with $n$. $f(n)=n\log n$. Is $n\log n=\Omega(n^{1+\epsilon})$? No — $n\log n$ is larger than $n$ only by a $\log n$ factor, not by any polynomial $n^\epsilon$. So neither Case 2 (not $\Theta(n)$) nor Case 3 (gap is not polynomial). **Master theorem does NOT apply.** (Aside: the actual answer is $\Theta(n\log^2 n)$, but you cannot get it from the textbook Master theorem.)

iv) $a=5,b=5$, $\log_5 5=1$, compare with $n$. $f(n)=n^2=\Omega(n^{1+\epsilon})$ (take $\epsilon=1$). Regularity: $a f(n/b)=5(n/5)^2=n^2/5\le c n^2$ with $c=1/5<1$. **Case 3**. $T(n)=\Theta(f(n))=\Theta(n^2)$.

**Answers.** (a) i Case 1, ii Case 2, iii does not apply, iv Case 3. (b) i $\Theta(n)$, ii $\Theta(n\log n)$, iv $\Theta(n^2)$.

## Problem 2 (10%) — Asymptotic notation true/false

**Problem.** State whether each is true or false.

**Type.** Decide $O,\Omega,\Theta,o,\omega$ claims. Recall: $o$ = strictly smaller (ratio $\to 0$), $\omega$ = strictly larger (ratio $\to\infty$), $O/\Omega$ allow equality. Constants don't matter for $O/\Omega/\Theta$.

**Method.** For each pair $f,g$: take $\lim f/g$. $\to 0$: $f=o(g)$ (and $O$, not $\Omega/\omega/\Theta$). $\to\infty$: $f=\omega(g)$ (and $\Omega$). $\to c\neq 0$: $f=\Theta(g)$ (so $O$ and $\Omega$, but not $o/\omega$). Keep only the highest-order term of polynomials.

**Worked solution.**
- i) $1=O(2)$. Both constants; $1\le c\cdot 2$. **True.**
- ii) $1=\Omega(2)$. Both constants; $1\ge c\cdot 2$ for small $c$. **True.**
- iii) $n=O(n^2)$. $n/n^2\to0$. **True.**
- iv) $n=\Omega(n^2)$. $n$ grows slower than $n^2$. **False.**
- v) $3x+2x^2+x^3=\Theta(x+2x^2+3x^3)$. Both are $\Theta(x^3)$. **True.**
- vi) $\log n=o(n/\log n)$. $\frac{\log n}{n/\log n}=\frac{\log^2 n}{n}\to0$. **True.**
- vii) $n^{1/2}=o(n/2^n)$. RHS $=n/2^n\to0$, LHS $\to\infty$. Ratio $\to\infty$, not $0$. **False.**
- viii) $\log n=\omega(\log n)$. A function is never $\omega$ of itself (ratio $=1$, not $\infty$). **False.**
- ix) $2^n\log n=\omega(2^n)$. Ratio $=\log n\to\infty$. **True.**
- x) $n^2/\log n=O(n(\log n)^2)$. Ratio $=\frac{n^2/\log n}{n\log^2 n}=\frac{n}{\log^3 n}\to\infty$. LHS bigger, so not $O$. **False.**

**Answers.** T, T, T, F, T, T, F, F, T, F.

## Problem 3 (7%) — Build-Max-Heap

**Problem.** Run Build-Max-Heap on $A=[2,1,5,4,8,6,7,9,3]$ (1-indexed, $n=9$). Give the array afterwards.

**Type.** Trace Build-Max-Heap (bottom-up Max-Heapify).

**Method.** For $i=\lfloor n/2\rfloor$ down to $1$, run Max-Heapify($A,i$): compare $A[i]$ with children $A[2i],A[2i+1]$; if a child is larger, swap with the largest child and recurse into that subtree. Indices above $\lfloor n/2\rfloor$ are leaves, skip them.

**Worked solution.** $n=9$, start at $i=4$.
- $i=4$: $A[4]=4$, children $A[8]=9,A[9]=3$. Largest child 9. Swap: ...9...4 → $A[4]=9,A[8]=4$. Leaf, stop.
- $i=3$: $A[3]=5$, children $A[6]=6,A[7]=7$. Largest 7. Swap: $A[3]=7,A[7]=5$. Leaf, stop.
- $i=2$: $A[2]=1$, children $A[4]=9,A[5]=8$. Largest 9. Swap: $A[2]=9,A[4]=1$. Now recurse at index 4: $A[4]=1$, child $A[8]=4$. Swap: $A[4]=4,A[8]=1$.
- $i=1$: $A[1]=2$, children $A[2]=9,A[3]=7$. Largest 9. Swap: $A[1]=9,A[2]=2$. Recurse at 2: $A[2]=2$, children $A[4]=4,A[5]=8$. Largest 8. Swap: $A[2]=8,A[5]=2$. Recurse at 5: leaf, stop.

**Answer.** $A=[9,8,7,4,2,6,5,1,3]$. (checked: Python)

## Problem 5 (7%) — Huffman coding

**Problem.** Frequencies: x:1000, y:200, z:600, æ:800, ø:300, å:400 (3300 chars). Build a Huffman tree; give a codeword per character and the total encoded length in bits.

**Type.** Build a Huffman tree (repeatedly merge two smallest), read off codes.

**Method.** Put all symbols in a min-priority queue by frequency. Repeatedly extract the two smallest, make a new internal node with their summed frequency, push it back. Label left edges 0, right edges 1; the code of a symbol is the root-to-leaf string. Total bits $=\sum_c \text{freq}(c)\cdot|\text{code}(c)|$ = sum of all internal-node frequencies.

**Worked solution.** Merges (smallest two each step):
1. y(200)+ø(300)=500.
2. å(400)+(500)=900. [next-smallest are 400 and 500]
3. z(600)+æ(800)=1400.
4. (900)+x(1000)=1900.
5. (1400)+(1900)=3300 = root.

One valid code assignment (exact bit-strings depend on tie/side choices; lengths are fixed):
- x: 2 bits, æ: 2, z: 2, å: 3, y: 4, ø: 4. E.g. x=11, z=00, æ=01, å=100, y=1010, ø=1011.

Total bits $=1000\cdot2 + 800\cdot2 + 600\cdot2 + 400\cdot3 + 200\cdot4 + 300\cdot4$
$=2000+1600+1200+1200+800+1200 = 8000$ bits.

(Equivalently sum of internal frequencies: $500+900+1400+1900+3300=8000$.)

**Answer.** Codeword lengths x,z,æ = 2; å = 3; y,ø = 4. Total = **8000 bits**. (checked: Python)

## Problem 7 (9%) — Counting binary trees

**7a (3%).** How many shapes of binary trees with 3 nodes? **Type:** Catalan number $C_n$. The number of binary tree shapes on $n$ nodes is $C_n=\frac{1}{n+1}\binom{2n}{n}$. $C_3=\frac{1}{4}\binom{6}{3}=\frac{20}{4}=5$. **Answer: 5.**

**7b (2%).** Trees with 3 nodes and keys 1,2,3 obeying max-heap order (parent $\ge$ children). **Method:** go through all $C_3=5$ shapes; for each, count labelings where every parent exceeds its children. The root is forced to be the max (3).

Go through all 5 shapes:
- Shape A: root, left child, left-grandchild (left chain). Forced 3→2→1. 1 way.
- Shape B: root, left child, that child's right child. Forced 3→2→1. 1 way.
- Shape C: root with two children (the one balanced shape). Root=3, children {1,2} in either order → 2 labelings, both heap-valid.
- Shape D: root, right child, its left grandchild. Forced 3→2→1. 1 way.
- Shape E: root, right child, its right grandchild (right chain). Forced 3→2→1. 1 way.

The 4 chain shapes each force 3>2>1 along the chain (1 labeling each = 4). The balanced shape gives 2. Total heap-valid = $4+2=6$.

**7b answer: 6.** (checked: Python) Pick the shape (5 ways), the root is forced to 3, and only the balanced shape leaves a free choice of which child gets 2 vs 1, adding one extra.

**7c (2%).** Trees with 3 nodes, keys 1,2,3 obeying inorder (binary-search-tree order). **Method:** a BST is determined by its shape, since inorder fixes which key sits at each position. So there is exactly one BST per shape. **Answer: 5** (= number of shapes $C_3$).

**7d (2%).** Trees with 3 nodes, keys 1,2,3 obeying BOTH inorder and max-heap order. **Method:** inorder $\Rightarrow$ BST (left<node<right); max-heap $\Rightarrow$ node$\ge$children. In a BST any node's right child is larger, contradicting max-heap unless there is no right child anywhere except... the root is both the BST "middle-ish" and must be the max. For both to hold simultaneously the tree must be a left-going chain 3→2→1 where 3 is root, 2 its left child, 1 its left child: inorder gives 1,2,3 (correct) and heap holds (3>2>1). Any right child breaks heap order. **Answer: 1.**

## Problem 8 (5%) — Loop invariant

**Problem.** `GangeSyv(n)`: x=n; r=0; while x>0 { x=x-1; r=r+7 }; return r. Computes $7n$. For each, is it a loop invariant (true every time the while-test is evaluated, for all integers $n\ge0$)?
- i) $r=7n$ · ii) $r<7n$ · iii) $n-x=7r$ · iv) $7x+r=7n$ · v) $x\ge0$

**Type.** Verify candidate loop invariants (true at every test of the guard).

**Method.** Check the predicate at each guard evaluation: initialization (first test, before any iteration) and after each iteration. After $k$ iterations: $x=n-k$, $r=7k$.

**Worked solution.** At the $k$-th guard test ($k=0,1,\dots,n$): $x=n-k$, $r=7k$, with $0\le k\le n$.
- i) $r=7n$? At $k=0$, $r=0$, fails unless $n=0$. **No.**
- ii) $r<7n$? At the final test $k=n$: $r=7n$, not $<7n$. **No.**
- iii) $n-x=7r$? $n-x=k$, $7r=49k$. Equal only if $k=0$. **No.**
- iv) $7x+r=7n$? $7(n-k)+7k=7n$. Holds for all $k$. **Yes** (this is the real invariant proving correctness).
- v) $x\ge0$? $x=n-k$ with $k\le n$, so $x\ge0$ at every test. **Yes.**

**Answers.** i No, ii No, iii No, iv Yes, v Yes.

## Problem 9 (8%) — Read off Θ running times

**Problem.** Give $\Theta$ running time in $n$ for each.

**Type.** Count loop iterations from pseudocode.

**Algoritme1:** `for i=1..n { j=i; while j>0 {j=j-1} }`. Inner runs $i$ times; total $\sum_{i=1}^n i=\frac{n(n+1)}{2}$. **$\Theta(n^2)$.**

**Algoritme2:** `i=1; while i<n { i=i*2 }`. $i$ doubles, stops after $\sim\log_2 n$ steps. **$\Theta(\log n)$.**

**Algoritme3:** `i=1; while i<n { i=i*n }`. First iteration sets $i=1\cdot n=n$, then $i<n$ is false. At most one iteration (zero if $n=1$). **$\Theta(1)$.**

**Algoritme4:** `i=1; while i<n { s=0; while s<n {s=s+i}; i=i*2 }`. Inner loop runs $\lceil n/i\rceil$ times for $i=1,2,4,\dots<n$. Total $\approx n+\tfrac n2+\tfrac n4+\dots = 2n$. **$\Theta(n)$.** (Simulated counts: n=100→201, n=1000→2000 — confirms $\Theta(n)$.)

## Problem 10 (12%) — Minimum number of squares (DP)

For $n\ge0$, $K(n)$ = fewest integers whose squares sum to $n$. Recurrence:
$K(0)=0$; $K(n)=1+\min\{K(n-a^2): a\ge1,\ a^2\le n\}$ for $n\ge1$.

**10a (3%).** Give $K(19)$ and a way to write 19 as that many squares.
Compute: $K(19)=3$, e.g. $19=1^2+3^2+3^2=1+9+9$. (Also $19=9+9+1$.) **$K(19)=3$.** (checked: Python)

**10b (3%).** DP algorithm computing $K(n)$, with time and space.
**Method/recipe.** Bottom-up table $\text{dp}[0..n]$.
```
K-DP(n):
  dp[0] = 0
  for m = 1 to n:
      best = +infinity
      a = 1
      while a*a <= m:
          best = min(best, 1 + dp[m - a*a])
          a = a + 1
      dp[m] = best
  return dp[n]
```
For each $m$ the inner loop runs $\lfloor\sqrt m\rfloor\le\sqrt n$ times.
**Time:** $\sum_{m=1}^n \sqrt m = \Theta(n\sqrt n)=\Theta(n^{3/2})$. **Space:** $\Theta(n)$ for the table.

**10c (3%).** Extension that also recovers $k=K(n)$ squares $a_1,\dots,a_k$.
Store, alongside dp, the minimizing $a$ for each $m$: `choice[m] = the a achieving the min`. After filling the table, reconstruct:
```
m = n; result = []
while m > 0:
    a = choice[m]
    result.append(a)
    m = m - a*a
```
This emits exactly $K(n)$ values whose squares sum to $n$. Extra time/space $\Theta(n)$ for the `choice` array and $\Theta(K(n))$ for the walk — does not change the overall $\Theta(n^{3/2})$ time / $\Theta(n)$ space. (For $n=19$ it yields $a=1,3,3$.)

**10d (3%).** Argue the recurrence is correct.
- **Base.** $K(0)=0$: the empty sum of squares equals 0, and you cannot do better than 0 terms.
- **Inductive step.** Let $n\ge1$ and suppose an optimal solution writes $n=a_1^2+\dots+a_k^2$ with $k=K(n)$, all $a_i\ge1$. Pick any one term, say $a_k$ (so $a_k^2\le n$). Removing it leaves $a_1^2+\dots+a_{k-1}^2=n-a_k^2$, a representation of $n-a_k^2$ using $k-1$ squares. Hence $K(n-a_k^2)\le k-1=K(n)-1$, i.e. $K(n)\ge 1+K(n-a_k^2)\ge 1+\min_a K(n-a^2)$.
- **Other direction.** For any valid $a$ ($a^2\le n$), take an optimal representation of $n-a^2$ (size $K(n-a^2)$) and append $a$: this represents $n$ with $1+K(n-a^2)$ squares, so $K(n)\le 1+K(n-a^2)$. Taking the min over $a$ gives $K(n)\le 1+\min_a K(n-a^2)$.
- Both inequalities give equality: $K(n)=1+\min\{K(n-a^2):a\ge1,a^2\le n\}$. The argument is the standard optimal-substructure / cut-and-paste argument: an optimal solution for $n$ contains an optimal solution for the subproblem $n-a^2$.
